Welcome to the **MarkdownLM Governance Rules** repository. This repository serves as the template for all technical constraints, architectural boundaries, style guides, and operational standards for the MarkdownLM ecosystem. 

By centralizing these rules into a machine-readable governance layer, AI agents and human developers can reliably build, scale, and maintain the platform without violating core invariants.


## Knowledge Base Categories

This repository provides ready-to-use templates for various categories to keep things simple for users.

- [**Architecture**](./knowledge/architecture/architecture.md): Layering, boundaries, and system design.
- [**Stack**](./knowledge/stack/stack.md): Authorized frameworks, libraries, and versions.
- [**Testing**](./knowledge/testing/testing.md): Test frameworks, required coverage, and testing patterns.
- [**Deployment**](./knowledge/deployment/deployment.md): CI/CD pipelines, platforms, and scripts.
- [**Security**](./knowledge/security/security.md): Authentication, validation, and secret management.
- [**Style**](./knowledge/style/style.md): Naming conventions, formatting, and file organization.
- [**Dependencies**](./knowledge/dependencies/dependencies.md): Approved and banned packages.
- [**Error Handling**](./knowledge/error_handling/error_handling.md): Exception hierarchies, logging protocols, and monitoring.
- [**Business Logic**](./knowledge/business_logic/business_logic.md): Domain rules, pricing logic, subscription constraints, and permission models.
- [**General**](./knowledge/general/general.md): Any additional guidelines that don't fit above.
